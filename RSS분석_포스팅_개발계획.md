# 구글 뉴스 RSS 분석 포스팅 기능 개발 계획

## 목표

`gui_app.py`의 **글쓰기(Post) 탭**에 구글 뉴스 RSS 기사 선택 → Gemini AI 분석 → 텍스트 파일 저장 기능을 추가한다.

---

## 기능 흐름

```
[RSS URL 입력] → [기사 불러오기] → [체크박스 선택] → [Gemini 분석] → [Desktop/lists/ 저장]
```

## 코드 변경 내역

### 수정 파일: `gui_app.py`

#### 1. `build_post_tab()` 개편
- 상단에 RSS 분석 섹션 추가
- 기존 프롬프트/버튼/로그 섹션은 하단 유지

#### 2. 신규 메서드 목록

| 메서드 | 역할 |
|--------|------|
| `build_rss_section(container)` | RSS UI 섹션(URL입력+체크박스목록) 구성 |
| `load_rss_articles()` | RSS URL에서 기사 목록 파싱 → 체크박스 표시 |
| `parse_rss_feed(url)` | urllib + ElementTree로 RSS XML 파싱 |
| `start_analysis_thread()` | 분석 작업을 별도 스레드로 실행 |
| `run_analysis(cfg, articles)` | Gemini 분석 호출 + 파일 저장 실행 |
| `generate_analysis_content(api_key, articles)` | Gemini AI에 분석 요청 → JSON 응답 파싱 |
| `save_analysis_to_file(title, content_str)` | Desktop/lists/ 에 .txt 파일 저장 |

#### 3. Gemini AI 시스템 프롬프트 (분석 전용)

```
당신은 뉴스 기사 분석 전문 에디터입니다.
제공된 기사들을 분석하여 반드시 아래 JSON 형식으로만 응답하세요.

{
  "title": "분석 포스트 제목 (흥미롭고 SEO 최적화된)",
  "subheadings": [
    {
      "title": "소제목 1",
      "analysis": "수치와 통계를 포함한 상세 분석 (300자 이상)"
    },
    ... (총 10개)
  ],
  "tags": ["태그1", "태그2", ..., "태그12+"]
}

분석 요구사항:
- 각 소제목 분석에는 반드시 수치/퍼센트/통계 데이터 포함
- 태그는 제목과 소제목에서 핵심 키워드 추출, 12개 이상
- 객관적 사실에 기반한 정확한 분석
- 전체 포스트 길이는 7000자 이상 ~ 8000자 이내로 작성해줘
```

## 사용 라이브러리

| 라이브러리 | 용도 | 설치 필요 |
|-----------|------|-----------|
| `urllib.request` | RSS URL HTTP 요청 | ❌ (표준) |
| `xml.etree.ElementTree` | RSS XML 파싱 | ❌ (표준) |
| `os`, `pathlib` | 파일/폴더 경로 처리 | ❌ (표준) |
| `re` | 파일명 특수문자 제거 | ❌ (표준) |

**추가 패키지 설치 없음** — Python 표준 라이브러리만 사용

## 작업 순서

1. `build_post_tab()` 레이아웃 분리 (스크롤 가능 구조로 변경)
2. `build_rss_section()` 구현 — URL 입력 + 체크박스 목록 (Canvas 스크롤)
3. `parse_rss_feed()` 구현 — 표준 라이브러리 RSS 파서
4. `load_rss_articles()` 구현 — 파싱 결과를 체크박스로 표시
5. `generate_analysis_content()` 구현 — Gemini 분석 프롬프트
6. `save_analysis_to_file()` 구현 — Desktop/lists/ 저장
7. `start_analysis_thread()` + `run_analysis()` 구현 — 비동기 실행
8. 실행 테스트
